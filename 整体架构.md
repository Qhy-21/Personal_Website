# 羌花吟个人网站 — 整体架构

## 

**Vue 3 单页应用（SPA），采用「静态站点 + Serverless 薄 API」的 Jamstack 架构，部署在 Vercel。**

---

## 架构总览

```
浏览器
  │
  ▼
Vercel CDN ── 静态资源 (dist/) ── Vue 3 SPA
  │               │
  │               ├── Vue Router (懒加载 6 个页面)
  │               ├── vue-i18n (中/英)
  │               └── 本地 JS 数据模块 (技能/项目/日记)
  │
  ├── /api/visitor ── Serverless Function ── Upstash Redis (访客计数)
  │
  └── Waline CDN ────────────────────────── 第三方评论系统
```

**不是**传统 Spring Boot 那种前后端分离——后端只有一个 Serverless 函数。

---

## 技术栈分层

| 层 | 技术 | 职责 |
|---|---|---|
| **前端框架** | Vue 3 (Composition API) | 组件化 UI |
| **构建工具** | Vite 5 | 开发热更新 + 生产打包 |
| **路由** | Vue Router (`createWebHistory`) | 6 个页面懒加载 |
| **国际化** | vue-i18n | 中文 / English |
| **后端 API** | Vercel Serverless Function | `api/visitor.js`：访客计数 + 地理分布 |
| **数据存储** | Upstash Redis (REST API) | 持久化访客数据，pipeline 原子操作 |
| **评论系统** | Waline (`@waline/client`) | 开源评论，无需自建后端 |
| **静态资源** | 本地 JS 模块 (`src/data/`) | 技能/项目/日记等数据写在前端代码里 |
| **部署平台** | Vercel | 自动构建 + CDN + Serverless |
| **域名** | qhy-alfie.work | 自定义域名 |

## 后端 API 设计

只有一个接口：

```
GET /api/visitor
→ { total, countries: [{ name, count }], recent: [...] }
```

**实现细节：**
- 通过 `x-forwarded-for` 获取真实 IP
- 调用 `ip-api.com` 解析地理位置（国家/城市）
- 使用 Upstash Redis pipeline 一次完成：`INCR`（计数+1） + `LPUSH`（写入列表） + `LTRIM`（保留最新120条） + `LRANGE`（取最近15条）
- Redis 不可用时自动回退到内存模式

---

## 路由设计

- **前端**：Vue Router `createWebHistory` 模式，无 `#` 的干净 URL
- **服务端**：`vercel.json` 配置 fallback 规则——所有非 `/api/` 且不含文件扩展名的请求都回退到 `index.html`，由前端接管路由
- 这样用户直接访问 `/about` 或刷新页面不会 404

---

## 为什么选这个架构

1. **更新频率低**：内容基本是静态的，没必要跑常驻后端服务
2. **成本接近零**：Vercel 免费额度完全够用，Upstash Redis 也有免费层
3. **部署简单**：`git push` → Vercel 自动构建部署，不需要管服务器
4. **静态走 CDN**：HTML/CSS/JS 全球 CDN 分发，访问速度快
5. **动态按需**：唯一的动态需求（访客计数）用 Serverless 函数按需执行，无需维护服务器

这套架构在业界叫 **Jamstack**（JavaScript + API + Markup）——静态站点为主体，动态能力通过 API 按需补充。
